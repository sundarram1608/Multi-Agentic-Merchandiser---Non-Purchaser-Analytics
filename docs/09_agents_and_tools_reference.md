# Agents and Tools Reference

Comprehensive catalogue of every agent and tool in the Merchandising AI
system. One row per component, with the exact call/return contract, retry
budget, and where the call enters and leaves.

Notes on the "Input node / Output node" columns: every LLM agent and every
tool is called exclusively by the **Orchestrator** (`agents/orchestrator.py`).
No agent ever calls another agent, and no agent ever calls a tool directly.
The Orchestrator is the only hub.

Notes on the "Max retries before Supervisor" column: this is the budget at
which the Orchestrator escalates to the Supervisor (`_supervisor_fallback`).
Two loops are exceptions and do NOT escalate to the Supervisor:

- The **viz sub-pipeline** — when its retries exhaust, the chart is
  silently dropped and the text answer ships without it.
- The **Writer ↔ groundedness loop** (hard block) — when its retries
  exhaust, the orchestrator replaces the unverified text with the
  fixed `GROUNDEDNESS_FAIL_TEXT` constant rather than asking the
  Supervisor to recover.

## LLM-call plumbing (not in the main table)

These live in `agents/llm.py` and are not "agents" or "tools" themselves —
they are the shared HTTP/SDK layer every LLM agent uses. Listed here for
completeness.

| Helper | Purpose |
|---|---|
| `call_llm(system, user, model, temperature, max_tokens)` | aisuite text completion entry point |
| `call_llm_json(...)` | Same, plus JSON parsing with defensive ` ```json ``` ` fence stripping |
| `call_llm_vision(system, user_text, image_png, ...)` | Multimodal call via the native `anthropic` SDK (aisuite's Anthropic vision support is patchy); base64-encodes the PNG internally |
| `call_llm_vision_json(...)` | Same plus JSON parsing |
| `get_client()` | Lazy aisuite `Client` singleton — one HTTP pool reused across all agents |
| `load_prompt(name)` | Read `agents/prompts/<name>.txt` from disk |

## Retry-budget summary (the same information condensed)

| Loop | Inner cap | Outer cap | What happens at exhaustion |
|---|---|---|---|
| Coder ↔ Code Reviewer (technical) | 5 | — | Supervisor (`_supervisor_fallback`) |
| Output Reviewer ↔ Coder (semantic) | — | 2 | Supervisor (`_supervisor_fallback`) |
| Writer ↔ groundedness (hard block) | 2 | — | Ship `GROUNDEDNESS_FAIL_TEXT` instead of unverified Writer text (no Supervisor) |
| Viz Coder ↔ Viz Code Reviewer (static) | 2 | — | Chart dropped, text answer still ships (no Supervisor) |
| Viz Reviewer revise round (multimodal) | — | 1 | Last rendered PNG ships, or chart dropped if verdict was `drop` |
| Supervisor `retry_with_strategy` | 1 one-shot Coder attempt | — | If it fails or returns nothing, Orchestrator returns the Supervisor's message |
